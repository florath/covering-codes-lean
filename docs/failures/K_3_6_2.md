# Failed attempts for `K_3(6,2)`

This note records private exploratory work around proving exactness for
`K_3(6,2)`.  The public Lean source contains an explicit radius-2 ternary
length-6 code of size `17`, so the remaining hard direction is ruling out a
16-center cover.

The experiments below were not certification artifacts.  They used prototype
Python/SciPy/OR-Tools scripts and produced no Lean, DRAT, or LRAT proof.  Their
purpose here is to mark approaches that were already tried and did not close
the case.

## Basic numbers

- Ambient space size: `3^6 = 729`.
- Radius-2 ball size: `1 + 6*2 + binom(6,2)*2^2 = 73`.
- A hypothetical 16-cover has total ball volume `16 * 73 = 1168`.
- Therefore it has `1168 - 729 = 439` excess hits.
- Radius-2 ball intersection sizes by center distance `0..6` are
  `[73, 33, 25, 12, 6, 0, 0]`.

## Overlap and local-distribution relaxations

The first route modeled a hypothetical 16-cover using only distance and local
multiplicity statistics.

Tried relaxations:

1. Pair-distance Delsarte LP with unordered pair counts `N_1,...,N_6`,
   Krawtchouk/Delsarte positivity, `sum N_i = binom(16,2)`, and the
   coverage-derived pair-overlap lower bound.
2. Global local-type second-moment LP.  Variables counted ambient points by
   local type `e = (e_0,...,e_6)`, where `e_i` is the number of centers at
   distance `i`.  Uncovered local types were omitted.  The model enforced first
   moments, second moments, and consistency with the pair-distance variables.
3. Integer versions of the global local-type model, tried with CP-SAT and
   HiGHS MILP.
4. Anchored first-moment LP after fixing `000000` and a normalized maximum
   weight anchor: `111110` for max-weight `5`, `111111` for max-weight `6`.
   Remaining centers were grouped by stabilizer orbits.
5. Anchored second-moment LP with ordered pair-orbit variables for the remaining
   14 centers, tying them to diagonal, upper-triangle, or full local second
   moments around the fixed zero/anchor pair.

Outcomes:

- The continuous pair-distance LP was feasible.  Its pair-overlap range was
  `528` to about `3197.333`.
- The continuous global local-type second-moment LP was feasible.  In the
  minimum-overlap solution it used distance distribution
  `N = [0, 0, 16, 56, 48, 0]` with overlap `528`.
- The global integer local-type model did not resolve:
  CP-SAT returned `UNKNOWN` after 300 seconds; HiGHS MILP hit a 300-second time
  limit without a primal solution or infeasibility proof.
- Anchored first-moment LPs were feasible for both remaining normalized cases:
  max-weight `5` with `1,229,440` point-type variables and `34` orbit variables,
  and max-weight `6` with `1,074,400` point-type variables and `26` orbit
  variables.
- Anchored second-moment LP attempts were inconclusive.  HiGHS IPM returned a
  solver error on the diagonal model, and HiGHS dual simplex was stopped after
  several minutes due to degeneracy.  The upper-triangle model was larger and
  not useful without further reduction or a better certificate-producing solver.

Conclusion: pair overlap, Delsarte positivity, global local second moments, and
the implemented anchored first/second-moment relaxations are too weak or too
degenerate in this form.  Re-running these unchanged is unlikely to help.

## Inverse non-center formulation

The second route modeled non-centers.  Let `x_c = 1` mean that word `c` is not a
center.  A radius-2 cover is equivalent to every radius-2 ball containing a
center, so in non-center variables:

```text
sum_{c in B(p,2)} x_c <= 72
```

for every point `p`.  Proving no 16-center cover in a normalized case is then
the infeasibility of

```text
sum_c x_c >= 729 - 16 = 713
```

together with fixed centers set to `x = 0` and forbidden heavier centers set to
`x = 1`.

Outcomes:

- Global inverse optimization with only `000000` fixed found objective `712`,
  matching the known 17-center upper construction.  The run improved the CP-SAT
  upper bound on non-centers to `717` before stopping.  This does not prove
  exactness.
- Normalized max-weight threshold checks:
  max-weight `4` was `INFEASIBLE` in about `0.13s`;
  max-weight `5` was `UNKNOWN` after 300 seconds;
  max-weight `6` did not resolve quickly and was stopped.
- Residual inverse LP after fixing `000000` and the normalized anchor gave lower
  bounds of `15.0`, about `9.4988`, and about `9.6006` additional centers for
  max-weights `4`, `5`, and `6`, respectively.  Only max-weight `4` is pruned by
  this.
- Third-orbit residual LP scans pruned `0/34` max-weight `5` cases and `0/26`
  max-weight `6` cases.  Bounds stayed around `9.18..9.52` against `13` slots.
- Short third-orbit inverse CP-SAT threshold checks with `5s` per case produced
  only `UNKNOWN` on sampled max-weight `5` cases before the scan was stopped.

Conclusion: the inverse formulation is cleaner than the cover formulation for
some searches and quickly eliminates max-weight `4`, but it did not close the
hard max-weight `5` and `6` cases.

## Structure check for the 17-code

The current 17-word code was tested for a simple affine explanation:

- affine rank is `6` after translation by any codeword;
- unordered pair distance distribution is `{3: 32, 4: 46, 5: 54, 6: 4}`;
- it is not a union of two ternary affine planes meeting in one point.

Thus the easy explanation `17 = 9 + 9 - 1` does not apply to this code.

## Better next directions

Useful future work should change the certificate strategy, not just rerun the
models above.  Plausible directions are:

- a stronger anchored second- or third-moment relaxation with a robust
  certificate-producing LP/SDP path;
- cube-and-conquer over fourth-center orbits, with UNSAT cubes emitted as
  DRAT/LRAT-checkable SAT certificates;
- a verified finite-search certificate using the zero/anchor/orbit
  decomposition.
